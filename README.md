# Tagged

I made this game because I wanted to answer a simple question:

Can a game built around a single mechanic still be genuinely fun?

Tagged is a local multiplayer game for two players. One player is "it" and the objective is exactly what you'd expect: tag the other player before they tag you.

There are no upgrades, no progression systems, no unlocks, and no complicated rules to learn. You launch the game, grab a friend, and start playing.

What surprised me is how quickly such a simple idea turns competitive.

## What is it?

Tagged is a small game built in Godot around the classic game of tag.

The entire game can be understood in less than a minute, which was part of the goal. I wanted something that anyone could pick up immediately without needing a tutorial or explanation.

Despite that simplicity, matches tend to become chaotic very quickly.

## Why I made it

A lot of games today are built around adding more systems, more mechanics, and more content.

I wanted to go in the opposite direction.

Instead of asking "what can I add?", I asked "how much can I remove while keeping the game fun?"

Tagged is the result of that idea.

## Running the project

Clone the repository:

```bash
git clone https://github.com/TheLinuxGuy-ssh/Tagged.git
```

Open the project in Godot and run it.

That's all there is to it.

## About the code

This isn't a large project, but that's part of the reason I decided to make it public.

Small projects are often easier to learn from because you can understand the entire codebase without spending hours navigating through dozens of systems.

If you're learning Godot, feel free to explore the project, experiment with it, break things, and rebuild them.

## Final thoughts

Sometimes the most memorable games come from a single idea executed well.

Tagged isn't trying to be the next big thing. It's just a small game about chasing your friend around and trying not to get tagged.

And somehow, that's enough.
